# MyFitPick Credit Purchase Architecture

MyFitPick is credit-only. There is no active subscription system, plan entitlement, recurring checkout, or Paystack path.

## Credit Packs

Credit packs are trusted server-side values from `lib/payments/packs.ts`.

| Pack | Credits | Price |
| --- | ---: | ---: |
| Starter | 50 | $4.99 |
| Popular | 150 | $9.99 |
| Pro | 400 | $19.99 |
| Creator | 1,000 | $39.99 |

The browser may submit only `packId`, provider choice, and USDT network where relevant. Price, currency, and credit quantity are never accepted from the client.

## Providers

Stripe uses hosted Checkout with `mode: "payment"`. Checkout line items use trusted `price_data` generated by the server. `payment_method_types` is intentionally not hardcoded, so Stripe can show eligible cards, Link, wallet options, and local methods according to Dashboard/account/device eligibility.

CoinPayments NEW uses hosted USDT invoices through the signed merchant API. The app does not use static wallet addresses, screenshots, or user-submitted transaction hashes as proof of payment.

## USDT Networks

Enable only networks confirmed in the CoinPayments merchant account. The intended public options are:

- USDT on Tron (`TRC20`) - recommended, very low fees.
- USDT on BNB Smart Chain (`BEP20`) - recommended, very low fees.
- USDT on Ethereum (`ERC20`) - recommended, higher fees.
- USDT on Solana - fast, lower network fees.

`COINPAYMENTS_USDT_NETWORK_ALLOWLIST` must include exact CoinPayments `providerCurrencyId` values before checkout can start.

## Fulfilment

Both providers grant Credits through `grantPurchasedCredits({ purchaseId, provider, providerReference })`.

The service loads trusted purchase values from `CreditPurchase`, validates the pack, creates one `CreditTransaction` using `credit-purchase:{purchaseId}`, increments the user wallet once, and marks the purchase `credited`.

The success redirect page only polls status. It never grants Credits.

## Webhook Idempotency

Every provider event is recorded in `ProcessedPaymentEvent` with a unique `{ provider, eventId }` index. Duplicate webhooks return success without duplicate wallet updates.

Stripe fulfils from paid Checkout session events and records PaymentIntent/refund/dispute events for reconciliation.

CoinPayments grants Credits only on terminal invoice completion after signature verification and invoice reconciliation.

## Refunds And Disputes

Refunds are admin-only. Full or partial refunds reverse purchased Credits only when enough purchased Credits remain. If the user already spent those Credits, the purchase is marked `review_required`.

Disputes mark purchases for review. Lost disputes attempt the same safe reversal rule.

## Database Indexes

- `CreditPurchase`: `userId + createdAt`, `status + createdAt`, unique Stripe Checkout Session ID, unique Stripe PaymentIntent ID, unique CoinPayments invoice ID, unique provider reference.
- `ProcessedPaymentEvent`: unique `provider + eventId`.
- `CreditTransaction`: unique `user + feature + referenceId` and unique `feature + referenceId`.

## Migration

Run `npm run migrate:credit-wallets` only after reviewing the target environment. The migration is idempotent, initializes missing wallet fields, unsets legacy subscription flags, preserves Credit balances, and does not drop old database collections.

## Environment Variables

```env
STRIPE_SECRET_KEY=
STRIPE_WEBHOOK_SECRET=
COINPAYMENTS_CHECKOUT_ENABLED=false
COINPAYMENTS_API_BASE_URL=https://a-api.coinpayments.net/api/v2
COINPAYMENTS_CLIENT_ID=
COINPAYMENTS_CLIENT_SECRET=
COINPAYMENTS_WEBHOOK_SECRET=
COINPAYMENTS_WEBHOOK_URL=https://YOUR_DOMAIN/api/webhooks/coinpayments
COINPAYMENTS_USD_CURRENCY_ID=
COINPAYMENTS_USDT_NETWORK_ALLOWLIST=
```

Do not commit live values.

For the Stripe-only production launch, keep `COINPAYMENTS_CHECKOUT_ENABLED=false`. USDT checkout remains visible in the wallet as coming soon and should only be enabled after CoinPayments has been configured and tested end to end.

## Production Activation

Stripe Dashboard:

- Activate the production account and complete business verification.
- Add payout bank account and statement descriptor.
- Enable cards, Link, Apple Pay, Google Pay, and eligible local methods.
- Add webhook endpoint `https://YOUR_DOMAIN/api/webhooks/stripe`.
- Configure webhook event delivery for Checkout, PaymentIntent, refund, and dispute events handled by the app.
- Test success, decline, 3D Secure, delayed methods where enabled, refunds, and disputes.

CoinPayments Dashboard:

- Complete merchant verification.
- Confirm API base URL and credentials for the assigned API instance.
- Confirm exact USDT network provider currency IDs.
- Register `https://YOUR_DOMAIN/api/webhooks/coinpayments`.
- Test invoice creation, expiry, underpayment, overpayment, completion, duplicate webhook replay, and reconciliation.
